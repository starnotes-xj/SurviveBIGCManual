# 梯子获取

访问 GitHub、Google、Claude、Docker Hub 等境外网站都需要梯子。这是学习信息安全的基础设施，越早配好越好。

---

## 基本概念

**机场** = 提供代理服务的服务商，你购买订阅后用客户端连接他们的服务器，流量经由境外服务器转发。

**客户端** = 运行在你电脑上的软件，负责把请求通过机场服务器转发出去。

常见协议：

| 协议 | 特点 |
|---|---|
| Shadowsocks (SS) | 老牌协议，稳定，部分地区已被封锁 |
| VMess / VLESS | V2Ray 系列，抗封锁能力强 |
| Trojan | 伪装成 HTTPS 流量，不易被识别 |
| Hysteria2 | 基于 UDP，速度快，适合看 4K 视频 |

## 配置订阅

购买机场后会得到一个**订阅链接**（URL），导入步骤：

1. 打开 Clash Verge Rev，点击左侧「**订阅**」
2. 在「订阅文件链接」中粘贴订阅 URL，点击「**导入**」
3. 回到「**代理**」页，选择一个节点（推荐新加坡）

**首次配置建议开启以下选项**（在「设置」中）：

| 选项 | 说明 |
|---|---|
| 系统代理 | 让浏览器等软件走代理，必须开启 |
| 开机自启 | 重启后自动运行，避免忘记开 |
| 局域网连接 | 允许同局域网其他设备共享代理 |
| DNS 覆写 | 防止 DNS 污染，建议开启 |
| 统一延迟 | 让延迟数据更准确 |
| 关闭 IPv6 | 避免部分网站走 IPv6 绕过代理 |

**代理模式说明：**

| 模式 | 适用场景 |
|---|---|
| 规则模式（推荐） | 国内直连、境外走代理，日常使用 |
| 全局模式 | 所有流量走代理，调试时用 |
| 直连模式 | 关闭代理，访问国内网站时切换 |

## 常见问题

**Q：重启电脑后网络访问不了？**

原因是系统代理还挂着，但 Clash 没有启动，导致流量被拦截。解决方法：

- **根治**：在设置中开启「**开机自启**」，重启后 Clash 自动运行
- **临时修复**：打开 Clash Verge Rev 即可恢复，或手动关闭系统代理（Windows 设置 → 网络和 Internet → 代理 → 关闭「使用代理服务器」）
- 不用梯子时切换到「**直连模式**」，关闭系统代理，这样即使软件没开也不影响上网

**Q：开了梯子但某些网站还是打不开？**

检查当前是否为「规则模式」，若目标网站不在规则列表中可临时切换「全局模式」。

**Q：GitHub clone 速度很慢？**

在终端单独配置 git 代理：

```bash
git config --global http.proxy http://127.0.0.1:7897
git config --global https.proxy http://127.0.0.1:7897
```

端口号（7897）以 Clash「设置 → 端口」显示的为准。

**Q：梯子开着但浏览器打不开？**

确认「系统代理」已打开，或安装浏览器扩展 [SwitchyOmega](https://github.com/FelisCatus/SwitchyOmega) 手动指定代理端口。

**Q：Windows 端口被占用怎么办？**

以管理员身份运行 PowerShell：

```powershell
# 查看 7897 端口占用
netstat -ano | findstr :7897
# 根据 PID 结束进程
taskkill /PID <PID号> /F
```
