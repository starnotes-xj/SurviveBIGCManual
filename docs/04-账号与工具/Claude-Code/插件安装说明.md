# Claude Code 插件安装说明

安装插件时有两个关键选项：**安装范围**（插件对哪些项目生效）和**插件来源**（从哪个市场安装）。

---

## 安装范围

安装范围通过 `--scope` 参数指定，决定插件作用于哪些项目。

=== "user（默认）"

    **用户级**，所有项目共享。

    - 存储位置：`C:\Users\用户名\.claude\plugins\`
    - 插件更新后对所有项目同时生效
    - 适合常用工具：代码审查、格式化、通用辅助等

    ```powershell
    claude plugin install hookify
    # 等同于
    claude plugin install --scope user hookify
    ```

=== "project"

    **项目级**，仅当前项目可用。

    - 存储位置：项目目录的 `.claude/plugins/`
    - 配置可提交到 git，团队成员共享
    - 适合项目专用插件：特定测试框架、项目规范检查等

    ```powershell
    claude plugin install --scope project hookify
    ```

=== "local"

    **本地临时**，仅当前会话可用。

    - 存储在临时目录，退出后不保留
    - 适合评估插件效果、一次性使用

    ```powershell
    claude plugin install --scope local hookify
    ```

!!! tip "怎么选"
    - 日常使用的插件 → `user`（默认，省略参数即可）
    - 团队项目的统一配置 → `project`（提交到 git 后团队共享）
    - 不确定好不好用 → `local` 先试，满意再正式安装

## 插件来源

插件名后加 `@市场名` 可指定来源，省略时自动从可用市场查找。

| 来源 | 说明 | 示例 |
|---|---|---|
| **官方市场** | Anthropic 维护，稳定可靠 | `claude-plugins-official` |
| **第三方市场** | 社区开发者维护，功能多样 | `ui-ux-pro-max-skill` |

```powershell
# 从官方市场安装
claude plugin install hookify@claude-plugins-official

# 从第三方市场安装
claude plugin install ui-ux-pro-max@ui-ux-pro-max-skill

# 查看已配置的市场
claude plugin marketplace list
```

## 常用命令

```powershell
# 查看已安装插件（含作用范围）
claude plugin list

# 查看插件详情
claude plugin info <插件名>

# 卸载插件
claude plugin uninstall <插件名>
```

---

## 常见问题

**Q：安装时提示找不到插件？**  
指定完整格式：`claude plugin install 插件名@市场名`

**Q：想换安装范围怎么办？**  
先卸载再重装：`claude plugin uninstall 插件名`，然后用新的 `--scope` 重新安装。

**Q：project 范围的插件如何同步给团队？**  
将 `.claude/plugins/` 目录提交到 git，团队成员 `git pull` 后自动获取。
