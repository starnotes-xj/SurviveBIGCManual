### 支付方式

ChatGPT Plus 支持直接填写 Visa/Mastercard 卡号，无需 PayPal：

1. 打开 [chat.openai.com](https://chat.openai.com)，点击左下角头像 → **Upgrade to Plus**
2. 在支付页面选择 **Credit / Debit Card**
3. 填写卡号、有效期（MM/YY）、CVV、账单地址

工商银行星座学生版 Visa 信用卡可直接使用，申请方式见 [工商银行 Visa 卡](../../05-校园生活/银行卡办理/工商银行Visa卡.md)。

## Codex CLI 快速入门

[Codex CLI](https://github.com/openai/codex) 是 OpenAI 出品的终端 AI 编程助手，在项目目录中直接对话，读文件、写代码、执行命令，全流程闭环。[oh-my-codex（OMX）](https://github.com/Yeachan-Heo/oh-my-codex) 是架在 Codex 之上的工作流增强层，提供更结构化的任务拆分、多角色协作和持久化状态管理。

!!! info "前提条件"
    - Node.js 20+
    - OpenAI API Key（或已有 Codex 订阅）
    - macOS / Linux（OMX 主要支持平台；Windows 建议用 WSL2）

### 安装

=== "已有 Codex CLI"

    ```bash
    codex --version          # 确认 Codex 可用
    npm install -g oh-my-codex
    omx setup
    ```

=== "从零开始（npm 管理）"

    ```bash
    npm install -g @openai/codex
    npm install -g oh-my-codex
    omx setup
    ```

!!! warning "不要合并安装"
    如果 Codex 已通过 Homebrew 安装，**不要**再用 `npm install -g @openai/codex oh-my-codex` 一并安装，会因 `EEXIST` 冲突报错。

配置 API Key：

```bash
export OPENAI_API_KEY="sk-..."
# 或写入 ~/.bashrc / ~/.zshrc 永久生效
```

验证安装：

```bash
omx doctor
codex login status
omx exec --skip-git-repo-check -C . "Reply with exactly OMX-EXEC-OK"
```

### 启动

```bash
omx --madmax --high    # 推荐（高并发 + 最大上下文）
omx --direct --yolo    # 不用 tmux，直接启动
```

### 核心工作流

OMX 标准三步：**澄清 → 规划 → 执行**，在 Codex 会话内输入：

```text
$deep-interview "说清楚你想做什么"
```
```text
$ralplan "批准最安全的实现方案"
```
```text
$ultragoal "把批准的方案拆成持久化 Codex 目标"
```

### 常用指令

| 指令 | 用途 |
|---|---|
| `$deep-interview "..."` | 澄清需求边界，任务模糊时先用这个 |
| `$ralplan "..."` | 审批实现方案和架构权衡 |
| `$ultragoal "..."` | 将批准方案转为持久化多目标任务 |
| `$ralph "..."` | 单 Owner 持续推进完成循环 |
| `$team "..."` | 并行协作，需要多角色协同时 |
| `$prometheus-strict "..."` | 高风险任务方案压测 |
| `/skills` | 浏览已安装的 OMX 技能 |

### 常用 CLI 命令

```bash
omx doctor               # 检查安装完整性
omx update               # 更新到最新版并刷新配置
omx setup                # 重新初始化
omx team status <name>   # 查看团队任务状态
omx explore --prompt "..." # 只读代码库查询
omx hud --watch          # 监控/状态面板
```

!!! warning "中文编码问题"
    实测在使用 Codex 编写 Markdown 文档时，**容易出现编码问题导致中文内容乱码或显示为问号**。建议让 Codex 写完后手动检查文件编码，或在任务描述里注明"文件使用 UTF-8 编码"。

!!! tip "用 Codex 订阅驱动 Claude Code"
    有 Codex 订阅但想用 Claude？参考 [用 Codex 订阅驱动 Claude Code](../Claude-Code/Codex订阅转Claude.md)，通过 `claude-code-proxy` 代理转换。
