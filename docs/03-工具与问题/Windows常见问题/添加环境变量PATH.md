# 添加程序到系统环境变量 PATH

在命令行中直接输入程序名（如 `claude`、`python`、`git`）就能运行，依赖的是系统的 **PATH 环境变量**。如果终端提示「不是内部或外部命令」，通常就是该程序所在目录没有加入 PATH。

以下以 **Claude Code** 为例说明操作步骤，其他程序同理。

---

## 第一步：找到程序所在目录

首先要知道程序安装在哪里。Claude Code 通过官方脚本安装后，可执行文件通常位于：

```
%USERPROFILE%\.local\bin
```

在 PowerShell 中确认位置：

```powershell
where.exe claude
```

如果有输出路径说明已在 PATH 中，无需继续；如果提示找不到，记下实际安装目录后继续下一步。

---

## 第二步：打开环境变量设置

**方法一：搜索**

按 `Win + S` 搜索「**编辑系统环境变量**」，点击打开。

**方法二：此电脑属性**

右键点击桌面上的「**此电脑**」→「**属性**」，在设备信息页面点击「**高级系统设置**」。

!!! tip "桌面没有此电脑图标？"
    在桌面空白处鼠标右键 →「**个性化**」→「**主题**」→「**桌面图标设置**」，勾选「**计算机**」并点击确定即可。

两种方式打开后，都点击右下角「**环境变量**」按钮，在下方「**系统变量**」区域找到 **Path**，双击打开。

---

## 第三步：添加程序目录

1. 点击「**新建**」
2. 粘贴程序所在目录的路径（例如 `C:\Users\你的用户名\.local\bin`）
3. 点击「**确定**」→「**确定**」→「**确定**」，关闭所有窗口

---

## 第四步：验证

**重新打开**一个 PowerShell 窗口（已打开的窗口不会读取新的 PATH），输入：

```powershell
claude --version
```

能正常输出版本号即配置成功。

!!! warning "必须重新打开终端"
    修改 PATH 后，**已经打开的终端不会生效**，必须关闭后重新打开才能识别新路径。

---

## 用户变量 vs 系统变量

| | 用户变量 | 系统变量 |
|---|---|---|
| 生效范围 | 仅当前 Windows 账号 | 所有账号 |
| 推荐场景 | 个人工具 | 全局共用工具 |

一般情况下添加到**系统变量**的 Path 即可。
